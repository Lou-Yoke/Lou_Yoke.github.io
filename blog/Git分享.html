    # 一、简介

    git是一个开源的分布式版本控制系统， 是Linux的老爹为了帮助管理Linux内核开发写的一个开放源码的版本控制工具，跟传统集中式版本管理工具svn相比最大的特点是，每个用户的本地仓库都存放着整个项目的代码，而svn则需要一个集中式的版本服务器。

    区分一下：

    1. git：一个开源的分布式版本控制工具
    2. github：一个使用git版本控制工具的全球最大开源代码托管平台
    3. bitbucket：也是一个代码托管平台，商业公司用比较多，支持每个用户免费创建5个私有仓库，而github私有仓库是要钱的
    4. gitlab：一个基于git的项目管理软件，很多公司常用来搭建内部私有代码仓库
      了解一下git：

    # 二、基本配置

    安装

    # Debian系安装  
sudo apt-get install git  
  
# macOS安装（先确保安装了homebrew）  
brew install git

    # 2. 配置账号

    ​
    ​ # 本着谁污染谁治理的原则，当你commit充满bug的代码的时候git必须提供必要的信息：
    ​ # 比如你是谁，怎么联系你
    ​ # 注意：当多个用户在同一个服务器上操作git时，配置的时候不要使用--global参数，毕竟你也不想为别人的bug背锅
    ​ git config --global user.name ''
    ​ git config --global user.email ''

    # 3. 创建仓库

    ​

    mkdir <project name>  
cd <project name>  
git init # 将一个目录初始化为一个代码仓库

    # 4. 克隆仓库

    git clone <https://github.com/ananan/project.git>

    # 三、常用操作

    # 1. 基本操作

    # 显示工作目录下已修改的文件  
git status  
  
# 显示与上次commit的版本相比改了啥  
git diff  
# 也可以查看具体分支的改动  
git diff <branch name>  
  
# 将修改的文件添加到stage区  
git add <target file>  
  
# 提交所有修改  
git commit -a  
# 说明一下改动了哪里是比较好的习惯  
git commit -m 'changed log'

    # 2. 同步到远程

    ​
    ​ # 列出当前配置的远程端仓库
    ​ git remote -v
    ​
    ​ # 显示远程仓库的信息
    ​ git remote show
    ​
    # 本地仓库与远程仓库关联起来
    git remote add origin http://github.com/ananan/test.git

    # 提交本地仓库到远程master分支，第一次提交到master分支时使用 -u 参数，git会把本地master分支和远程  
# master分支关联起来，在以后的push和pull会方便点  
git push -u origin master

    # 3. 更新到本地

    # 将当前远程分支的代码拖到本地  
git pull  
  
# 只拖远程代码，不合并到本地HEAD中  
git fetch --all  
  
# 当远程与本地的代码发生冲突的时候强制使用远端代码覆盖本地（慎用）  
git fetch --all  
git reset --hard origin/master

    # 4. 分支管理

    分支的理念是版本管理的核心，应该多利用分支

    # 查看本地分支  
git branch  
# 查看项目所有分支  
git branch -a  
  
# 在当前分支的基础之上创建并切换到新分支  
git branch -b <new branch name>  
  
# 删除本地分支  
git branch -d <branch name>  
  
# 删除远程分支  
git push <remote branch name> --delete

    # 5. 标签管理

    # 查看当前仓库的标签  
git tag  
  
# 给当前的版本打个标签，比如发布代码的时候搞个版本号  
git tag <tag name>  
# 打标签的时候说明一下是比较好的习惯  
git tag -a <tag-name> -m 'tag log'  
  
# 推送标签到远程  
git push origin <tag-name>

    # 6. 合并与撤销

    # 将分支合并到当前的HEAD中  
git merge <branch>  
  
# 合并两个分支  
git merge master dev  
  
# 放弃工作目录的修改  
git reset --hard HEAD

    # 四、进阶

    # 1.添加忽略文件

    在将项目推送到远程的时候最好屏蔽掉与项目无关的文件或者文件夹，可以在项目目录中添加 “.gitignore” 文件来实现。

    # 新建一个.gitignore 文件  
vim .gitignore  
  
# 在文件中添加你要忽略的文件，格式类似正则，忽略实例如下  
# 忽略指定文件  
.idea  
  
# 忽略类型文件  
*.jpg  
*.png  
  
# a.png 指定文件除外  
!a.png  
  
# 忽略指定目录  
src/

    # 2. 冲突解决

    分支冲突在git中是很常见的情况，当要从远程仓库拉取代码时会可能与本地分支会冲突，推送的时候也可能发生冲突。

    # 强制使用远程代码覆盖本地，先fetch远程代码，在不做任何的合并的情况下使用git reset 把HEAD指向刚刚下载的最新的版本  
git fetch --all  
git reset --hard origin/master  
  
# 将本地分支强制推送到远程，加上-f参数，但不推荐  
git push origin master -f  
  
# 大多数情况其实可以避免冲突的发生  
# 1. 在修改代码之前先git pull一下  
# 2. 将业务代码进行划分，尽量不要多个人同时修改同一个文件  
# 3. 通过gitflow工作流提升git流程效率，一定程度避免冲突的发生

    # 五、总结

    版本控制是一个非常大的话题，git的分布式和工作流都是非常好的设计，可惜我目前对git的了解也仅停留在初步的使用，之前朋友推荐了一本书pro- git的书，看了几章，非常收益。不过我一直认为，对于工具类的学习，大多数情况是按需而学，刻意去学而不用，很快也会忘掉，所以以后还是多多使用才行。
